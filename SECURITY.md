Security Policy
Reporting a Vulnerability
Potential security issues should not be reported via public GitHub issues.
Please use one of the following channels:
GitHub Private Vulnerability Reporting
Email: security@logosblockchain.com
Reports should include:
Clear description of the issue and its potential impact
Affected component and version (if known)
Minimal reproduction steps (without including secrets or private data)
Relevant logs or request samples (sanitized)
Receipt of valid reports will be acknowledged within a reasonable timeframe.
Coordinated Disclosure
LOGOS follows coordinated vulnerability disclosure principles.
We request reasonable time for investigation and remediation prior to any public disclosure.
Uncoordinated public disclosure may expose users, integrators, or infrastructure operators to avoidable risk.
Scope of This Policy
This policy applies exclusively to materials published within this GitHub repository.
In Scope
Public API surface
SDK behavior
Public documentation artifacts
Integration examples
Non-production infrastructure templates included in this repository
Out of Scope (Without NDA)
The following components are proprietary and not subject to public security review:
Core execution engine implementation
Consensus and finality mechanisms
Internal control or protection logic
Security-critical parameters or configurations
Production deployment topology
Administrative, debug, or partner-only endpoints
Security review of proprietary components requires formal agreement.
Partner Security Review
Qualified enterprise partners may request a controlled security review process under NDA.
This may include:
Access to extended technical documentation
Controlled inspection sessions
Coordinated third-party audit procedures
